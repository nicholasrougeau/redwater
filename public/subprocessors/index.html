Subprocessors

The third party providers we use to run our website and build software for clients. Last updated July 8, 2026.

Redwater Revenue uses the third party providers below to deliver our website and the software and automations we build for clients. Each processes data only to provide services to us and is bound by its own data processing agreement.

ProviderPurposePersonal data handled
SupabaseDatabase, authentication, and storage (website leads and client apps)Stored application data
VercelWebsite and app hostingData in transit, request logs
GoogleCalendar scheduling, Workspace, and sign in (OAuth)Contact and scheduling data, email
ConvertKitNewsletter emailSubscriber email
Plausible AnalyticsPrivacy friendly website analyticsAggregate usage data (cookieless, no personal data stored)
ResendTransactional and notification emailRecipient email and content
TwilioSMS and voice in client buildsPhone numbers, message content
CallRailCall tracking and recordingCaller numbers, recordings, transcripts
StripeDeposit and payment processingPayment and identity data
SquarePayment processingPayment data
OpenAIAI text generation in client toolsData submitted in prompts (no training on API data)
Anthropic (Claude)AI features in client toolsData submitted in prompts (no training on API data)
ModalCompute and proposal signing hubSigner name, title, timestamp; scheduled syncs
InstantlyCold email outreach (our own marketing)Prospect names and emails
ApifyBusiness listing scraping (lead generation)Publicly listed business contact data
Anymail FinderEmail enrichment and lookup (lead generation)Business email lookup
PandaDocProposals and e-signature (legacy)Signer contact and signature
PostizSocial analytics (internal dashboard)Aggregate social metrics (no customer personal data)

Not every client engagement uses every provider. The Data Processing Addendum for a given project lists the subprocessors that apply to it. We update this page when we add or remove a provider.
